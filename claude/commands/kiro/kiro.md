# /kiro - Spec-Driven Development Command

## Overview

「コードを書く前に構造を与える」 - Kiro の革新的 spec-driven development に完全準拠

従来のコード生成ツールとは異なり、開発の混沌に構造を与えることに重点を置いた Kiro の仕様駆動開発を実現。わずか 4 行程度の要件入力から、プロダクトマネージャーレベルの詳細な仕様と実装可能な設計まで段階的に展開し、**プロトタイプから本番環境**まで一貫した品質を保証します。

## Command Syntax

```bash
/kiro [機能説明]
```

## Examples

### Basic Usage
```bash
/kiro ユーザー管理システム
/kiro 商品レビュー機能
/kiro 決済システム API
```

### With Requirements
```bash
/kiro チャット機能 - WebSocket、リアルタイム通知、履歴管理を含む
/kiro EC サイトの在庫管理 - 商品追加、在庫更新、アラート機能を含む
```

## Workflow Phases

### Phase 1: Requirements Discovery
- 最小限の入力から詳細なユーザーストーリーを自動生成
- EARS 記法による構造化要件記述
- セキュリティ要件・エッジケースの自動補完
- 対話的な要件の精緻化
- **Output**: `requirements.md`

### Phase 2: Design Exploration
- コードベース分析と最適アーキテクチャ提案
- Mermaid 図による視覚的設計
- TypeScript インターフェース自動生成
- 技術選択肢の提示と議論
- **Output**: `design.md`

### Phase 3: Implementation Planning
- ベストプラクティス自動適用
- 品質チェックポイント自動設定
- 依存関係とリスク軽減の分析
- 実装順序の最適化
- **Output**: `tasks.md`

## Control Commands

- **「次に進んで」**: 現在のフェーズを完了してファイル生成、次フェーズへ
- **「修正して」**: 現在のフェーズ内で内容を調整・改善
- **「やり直して」**: 現在のフェーズを最初からやり直し
- **「詳しく説明して」**: より詳細な説明や選択肢を提示
- **「スキップして」**: 現フェーズをスキップして次へ（非推奨）

## EARS Notation

```
WHEN [状況・トリガー]
GIVEN [前提条件]
THEN [システムの動作]
AND [追加の動作]

WHILE [状態・プロセス]
THE SYSTEM SHALL [必須動作]
AND [関連動作]

WHERE [機能・コンポーネント]
IS [条件・状態]
THE SYSTEM SHALL [対応動作]
```

## Proven Results

- **2 日間でセキュアファイル共有アプリ**: 暗号化対応の本番レベルアプリケーション
- **1 晩でゲーム開発**: 未経験者が 2D パズルゲーム作成完了
- **週末でプロトタイプ→本番**: EC サイト商品管理システムの完成

## Best Practices

1. **段階的承認**: 各フェーズ完了時に承認を得る
2. **要件の明確化**: 曖昧さを解消してから次フェーズへ
3. **設計レビュー**: 実装前の設計確認
4. **品質基準**: テスト・アクセシビリティを含む包括的基準

## Differences from /plan

| 特徴 | /plan | /kiro |
|------|-------|-------|
| 対象 | 一般的な実装計画 | 機能仕様駆動開発 |
| 出力 | 単一計画文書 | 3 つの独立ファイル |
| 要件 | 基本的整理 | EARS 記法詳細基準 |
| 設計 | 技術選定中心 | コードベース分析ベース |
| 品質 | 基本テスト戦略 | 包括的品質要件 |
