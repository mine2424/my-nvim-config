# 統合キーバインディングガイド

## 📋 概要

このドキュメントでは、Neovim、Tmux、WezTermの統一されたキーバインディング設計を説明します。
3つのツール間で一貫性のある操作感を実現し、効率的な開発環境を提供します。

## 🎯 設計原則

### 1. 一貫性
- **同じ操作は同じキー**: ペイン移動は常に `Ctrl+h/j/k/l`
- **階層的な設計**: WezTerm → Tmux → Neovim の順で処理
- **衝突の回避**: 各レイヤーで適切にキーを使い分け

### 2. 直感性
- **Vim風**: `h/j/k/l` による方向指定
- **覚えやすい**: Leader + 意味のある文字（`f` = find, `g` = git）
- **モード明確**: どのツールが反応するか明確

### 3. 効率性
- **最小キーストローク**: よく使う操作は短いキーで
- **ホームポジション**: 手を動かさずに操作可能
- **モディファイアキーの統一**: `Ctrl`, `Alt`, `Leader` の使い分け

## 🔑 キー配置の全体像

```
┌─────────────────────────────────────────────────────────────┐
│ WezTerm (最外層)                                             │
│ - タブ管理: Ctrl+Shift+[数字/T/W]                            │
│ - ペイン分割: Ctrl+Shift+[|/-]                               │
│ - 設定: Ctrl+Shift+,                                         │
│                                                              │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ Tmux (中間層)                                            │ │
│ │ - Prefix: Ctrl+A                                         │ │
│ │ - ウィンドウ管理: Prefix + [c/n/p/数字]                  │ │
│ │ - ペイン分割: Prefix + [|/-]                             │ │
│ │ - セッション: Prefix + [s/d]                             │ │
│ │                                                          │ │
│ │ ┌────────────────────────────────────────────────────┐  │ │
│ │ │ Neovim (最内層)                                     │  │ │
│ │ │ - Leader: Space                                     │  │ │
│ │ │ - ファイル操作: Leader + f[f/g/b/r]                │  │ │
│ │ │ - Git操作: Leader + g[s/c/p/b]                     │  │ │
│ │ │ - LSP: gd/gr/K/Leader+[ca/rn/f]                    │  │ │
│ │ │ - ウィンドウ: Leader + [s/w][v/h/e/x]              │  │ │
│ │ └────────────────────────────────────────────────────┘  │ │
│ └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘

統合ナビゲーション: Ctrl+h/j/k/l (全レイヤーで共通)
```

## 🖥️ WezTerm キーバインディング

> **💡 OS別のキー**: macOSは`Cmd`、Windows/Linuxは`Ctrl`を使用

### タブ管理

| キー (macOS) | キー (Win/Linux) | 動作 | 説明 |
|-------------|-----------------|------|------|
| `Cmd+T` | `Ctrl+T` | 新規タブを作成 | 現在のペインと同じドメインで新しいタブを開く |
| `Cmd+W` | `Ctrl+W` | タブを閉じる | 確認ダイアログを表示してからタブを閉じる |
| `Cmd+[1-9]` | `Ctrl+[1-9]` | タブ番号で切り替え | 1-8は該当番号、9は最後のタブへ |
| `Cmd+Tab` | `Ctrl+Tab` | 次のタブへ移動 | 右隣のタブに切り替える |
| `Cmd+Shift+Tab` | `Ctrl+Shift+Tab` | 前のタブへ移動 | 左隣のタブに切り替える |
| `Cmd+Shift+PageUp` | `Ctrl+Shift+PageUp` | タブを左に移動 | タブの順序を変更 |
| `Cmd+Shift+PageDown` | `Ctrl+Shift+PageDown` | タブを右に移動 | タブの順序を変更 |

### ペイン管理

| キー | 動作 | 説明 |
|------|------|------|
| `Ctrl+Shift+\|` | 垂直分割 | 現在のペインを左右に分割 |
| `Ctrl+Shift+_` | 水平分割 | 現在のペインを上下に分割 |
| `Leader x` | ペインを閉じる | 確認ダイアログを表示してからペインを閉じる |
| `Leader z` | ペインズーム | 現在のペインを最大化/復元 |
| `Leader r` | リサイズモード | ペインリサイズモードに入る（`h/j/k/l`でリサイズ） |

### ナビゲーション

| キー | 動作 | 説明 |
|------|------|------|
| `Ctrl+h` | 左のペインへ移動 | Tmux/Neovimと統合されたナビゲーション |
| `Ctrl+j` | 下のペインへ移動 | Tmux/Neovimと統合されたナビゲーション |
| `Ctrl+k` | 上のペインへ移動 | Tmux/Neovimと統合されたナビゲーション |
| `Ctrl+l` | 右のペインへ移動 | Tmux/Neovimと統合されたナビゲーション |

### コピーモード

| キー | 動作 | 説明 |
|------|------|------|
| `Leader [` または `Leader v` | コピーモード開始 | スクロールバックを閲覧・コピーするモードに入る |
| `v` (コピーモード内) | セル選択開始 | 文字単位で選択範囲を指定 |
| `V` (コピーモード内) | 行選択開始 | 行単位で選択範囲を指定 |
| `Ctrl+v` (コピーモード内) | 矩形選択 | 矩形範囲で選択 |
| `y` (コピーモード内) | コピーして終了 | 選択範囲をクリップボードにコピーしてモードを終了 |
| `Esc` または `q` (コピーモード内) | コピーモード終了 | コピーせずにモードを終了 |

### その他

| キー (macOS) | キー (Win/Linux) | 動作 | 説明 |
|-------------|-----------------|------|------|
| `Cmd+Shift+R` | `Ctrl+Shift+R` | 設定リロード | wezterm.luaの変更を反映 |
| `Cmd+Shift+P` | `Ctrl+Shift+P` | コマンドパレット | 利用可能なコマンドを検索・実行 |
| `Cmd+F` | `Ctrl+F` | 検索 | 現在の選択範囲または空文字列で検索 |
| `Cmd++` / `Cmd+=` | `Ctrl++` / `Ctrl+=` | フォント拡大 | フォントサイズを大きくする |
| `Cmd+-` | `Ctrl+-` | フォント縮小 | フォントサイズを小さくする |
| `Cmd+0` | `Ctrl+0` | フォントリセット | フォントサイズをデフォルトに戻す |

## 🪟 Tmux キーバインディング

> **💡 Prefixキー**: `Ctrl+A` がPrefixキーです。すべてのTmuxコマンドは `Prefix + キー` の形式です。

### 基本操作

| キー | 動作 | 説明 |
|------|------|------|
| `Ctrl+A` | Prefixキー | すべてのTmuxコマンドの前に押す |
| `Prefix r` | 設定リロード | ~/.tmux.confの変更を反映 |
| `Prefix ?` | キーバインド一覧 | 利用可能なすべてのキーバインドを表示 |
| `Prefix :` | コマンドプロンプト | Tmuxコマンドを直接入力 |

### ペイン分割

| キー | 動作 | 説明 |
|------|------|------|
| `Prefix \|` | 垂直分割 | 現在のペインを左右に分割（`|`の形が縦線） |
| `Prefix -` | 水平分割 | 現在のペインを上下に分割（`-`の形が横線） |

### ペイン移動

| キー | 動作 | 説明 |
|------|------|------|
| `Ctrl+h` | 左のペインへ移動 | Neovimと統合されたナビゲーション |
| `Ctrl+j` | 下のペインへ移動 | Neovimと統合されたナビゲーション |
| `Ctrl+k` | 上のペインへ移動 | Neovimと統合されたナビゲーション |
| `Ctrl+l` | 右のペインへ移動 | Neovimと統合されたナビゲーション |
| `Prefix h` | 左のペインへ移動 | Prefixモードでの移動（フォールバック） |
| `Prefix j` | 下のペインへ移動 | Prefixモードでの移動（フォールバック） |
| `Prefix k` | 上のペインへ移動 | Prefixモードでの移動（フォールバック） |
| `Prefix l` | 右のペインへ移動 | Prefixモードでの移動（フォールバック） |
| `Prefix q` | ペイン番号表示 | 各ペインに番号を表示して選択可能にする |

### ペインリサイズ

| キー | 動作 | 説明 |
|------|------|------|
| `Prefix H` | 左に拡大 | 左方向に5ピクセル拡大（繰り返し可能） |
| `Prefix J` | 下に拡大 | 下方向に5ピクセル拡大（繰り返し可能） |
| `Prefix K` | 上に拡大 | 上方向に5ピクセル拡大（繰り返し可能） |
| `Prefix L` | 右に拡大 | 右方向に5ピクセル拡大（繰り返し可能） |

### ペイン操作

| キー | 動作 | 説明 |
|------|------|------|
| `Prefix x` | ペインを閉じる | 現在のペインを終了（確認なし） |
| `Prefix z` | ペインズーム | 現在のペインを最大化/復元 |
| `Prefix Space` | レイアウト変更 | 次のレイアウト（even-horizontal, even-vertical, main-horizontal, main-vertical, tiled）に切り替え |

### ウィンドウ管理

| キー | 動作 | 説明 |
|------|------|------|
| `Prefix c` | 新規ウィンドウ | 現在のペインのディレクトリで新しいウィンドウを作成 |
| `Prefix n` | 次のウィンドウ | 右隣のウィンドウに移動 |
| `Prefix p` | 前のウィンドウ | 左隣のウィンドウに移動 |
| `Prefix w` | ウィンドウ一覧 | すべてのウィンドウを表示して選択可能にする |
| `Prefix ,` | ウィンドウ名変更 | 現在のウィンドウの名前を変更 |
| `Prefix &` | ウィンドウを閉じる | 現在のウィンドウを終了（確認あり） |
| `Prefix [0-9]` | ウィンドウ番号で移動 | 指定した番号のウィンドウに直接移動 |

### セッション管理

| キー | 動作 | 説明 |
|------|------|------|
| `Prefix s` | セッション一覧 | すべてのセッションを表示して選択可能にする |
| `Prefix d` | デタッチ | セッションから切り離す（セッションはバックグラウンドで実行継続） |
| `Prefix $` | セッション名変更 | 現在のセッションの名前を変更 |

### セッション・ウィンドウ・ペインの終了

| 操作 | キー | 動作 | 説明 |
|------|------|------|------|
| **ペインを閉じる** | `Prefix x` | 現在のペインを終了 | ペイン内のプロセスも終了 |
| **ウィンドウを閉じる** | `Prefix &` | 現在のウィンドウを終了 | ウィンドウ内のすべてのペインが閉じられる |
| **セッションからデタッチ** | `Prefix d` | セッションから切り離す | セッションはバックグラウンドで継続 |
| **セッションを完全に終了** | 最後のウィンドウを閉じる | セッションが自動的に終了 | すべてのウィンドウを閉じるとセッションも終了 |
| **セッションを強制終了** | `tmux kill-session` | コマンドで終了 | ターミナルから実行（セッション名指定可） |

### コピーモード（vi風）

| キー | 動作 | 説明 |
|------|------|------|
| `Prefix [` | コピーモード開始 | スクロールバックを閲覧・コピーするモードに入る |
| `v` (コピーモード内) | 選択開始 | 文字単位で選択範囲を指定 |
| `V` (コピーモード内) | 行選択 | 行単位で選択範囲を指定 |
| `Ctrl+v` (コピーモード内) | 矩形選択 | 矩形範囲で選択 |
| `y` (コピーモード内) | コピーして終了 | 選択範囲をクリップボードにコピーしてモードを終了 |
| `Y` (コピーモード内) | 行全体をコピー | 現在の行全体をコピーして終了 |
| `Prefix ]` | ペースト | コピーした内容を貼り付け |

## 📝 Neovim キーバインディング

> **💡 Leaderキー**: `Space` がLeaderキーです。多くのNeovimコマンドは `<leader>キー` の形式です。

### 基本操作

| キー | 動作 | 説明 |
|------|------|------|
| `Space` | Leaderキー | カスタムキーバインドのプレフィックス |
| `Ctrl+S` | 保存 | 現在のファイルを保存（ノーマル/インサートモード） |
| `<leader>w` | 保存 | 現在のファイルを保存 |
| `<leader>q` | 終了 | 現在のウィンドウを閉じる |
| `<leader>Q` | 強制終了 | 保存せずに終了 |
| `<leader>x` | 全て保存して終了 | すべてのバッファを保存してNeovimを終了 |
| `jk` または `jj` | インサートモード脱出 | インサートモードからノーマルモードへ |

### クリップボード

| キー | 動作 | 説明 |
|------|------|------|
| `<leader>y` | システムクリップボードへヤンク | 選択範囲をシステムクリップボードにコピー |
| `<leader>Y` | 行をシステムクリップボードへヤンク | 現在の行をシステムクリップボードにコピー |
| `<leader>p` | システムクリップボードからペースト | システムクリップボードの内容を貼り付け |
| `<leader>P` | システムクリップボードから前へペースト | カーソル前へシステムクリップボードの内容を貼り付け |
| `<leader>d` | 削除（レジスタを汚さない） | 削除してもレジスタに保存しない |

### ナビゲーション

| キー | 動作 | 説明 |
|------|------|------|
| `Ctrl+d` | 半ページ下へスクロール | 画面中央を維持しながら下へスクロール |
| `Ctrl+u` | 半ページ上へスクロール | 画面中央を維持しながら上へスクロール |
| `n` | 次の検索結果 | 検索結果の次のマッチへ移動（画面中央維持） |
| `N` | 前の検索結果 | 検索結果の前のマッチへ移動（画面中央維持） |
| `H` | 行頭へ移動 | 行の先頭（空白を除く）へ移動 |
| `L` | 行末へ移動 | 行の末尾へ移動 |

### ウィンドウ移動（Tmux統合）

| キー | 動作 | 説明 |
|------|------|------|
| `Ctrl+h` | 左のウィンドウ/ペインへ移動 | NeovimウィンドウまたはTmuxペインへ移動 |
| `Ctrl+j` | 下のウィンドウ/ペインへ移動 | NeovimウィンドウまたはTmuxペインへ移動 |
| `Ctrl+k` | 上のウィンドウ/ペインへ移動 | NeovimウィンドウまたはTmuxペインへ移動 |
| `Ctrl+l` | 右のウィンドウ/ペインへ移動 | NeovimウィンドウまたはTmuxペインへ移動 |
| `Ctrl+\` | 前のペインへ移動 | 前回アクティブだったペインへ戻る |

### ウィンドウ管理

| キー | 動作 | 説明 |
|------|------|------|
| `<leader>sv` | 垂直分割 | 現在のウィンドウを左右に分割 |
| `<leader>sh` | 水平分割 | 現在のウィンドウを上下に分割 |
| `<leader>se` | サイズ均等化 | すべてのウィンドウのサイズを均等にする |

| `Ctrl+→` | 幅を増やす | ウィンドウの幅を2列増やす |

### バッファ管理

| `<leader>bd` | バッファを閉じる | 現在のバッファを閉じる |
| `<leader>bD` | バッファを強制閉じる | 保存せずにバッファを閉じる |

| `<leader>k` | 行を上に複製 | 現在の行を上に複製 |

### LSP（Language Server Protocol）

| キー | 動作 | 説明 |
|------|------|------|
| `gd` | 定義へジャンプ | シンボルの定義場所へ移動 |

## 🚀 クイック起動コマンド

### dev コマンド

Neovim + Tmuxを一発で起動するコマンド。Claude Codeとの併用を考慮した複数のレイアウトを提供。

#### 基本的な使い方

```bash
# カレントディレクトリで起動
dev

# セッション名を指定
dev myproject

# ディレクトリを指定
dev myproject ~/code/myapp

# レイアウトを指定
dev --layout claude    # Claude Code併用（デフォルト）
dev --layout split     # 分割レイアウト
dev --layout full      # フルスクリーン
```

#### レイアウト

**claude（デフォルト）**: Claude Code併用レイアウト
```
┌──────────┬──────┐
│          │ Git  │
│ Neovim   ├──────┤
│ (60%)    │ Term │
└──────────┴──────┘
```

**split**: エディタ + ターミナル分割
```
┌─────────────────┐
│ Neovim (70%)    │
├─────────────────┤
│ Terminal (30%)  │
└─────────────────┘
```

**full**: エディタフルスクリーン
```
┌─────────────────┐
│                 │
│ Neovim (100%)   │
│                 │
└─────────────────┘
```

#### インストール

```bash
# スクリプトをダウンロード（dotfilesリポジトリから）
cp scripts/dev ~/.local/bin/dev
chmod +x ~/.local/bin/dev
```

#### Claude Code併用のワークフロー

1. **WezTermで2タブ構成**
   - タブ1: `dev --layout claude` でNeovim起動
   - タブ2: `cursor .` でClaude Code起動

2. **OS標準のウィンドウ分割**
   - 左半分: WezTerm (Neovim + Tmux)
   - 右半分: Claude Code

3. **外部モニター使用時**
   - モニター1: WezTerm (フルスクリーン)
   - モニター2: Claude Code (フルスクリーン)

## 🔧 カスタマイズガイド

### 自分用のキーバインドを追加

**Neovim**
```lua
-- ~/.config/nvim/lua/config/keymaps.lua
keymap("n", "<leader>custom", "<cmd>YourCommand<cr>", { desc = "Your description" })
```

**Tmux**
```bash
# ~/.tmux.conf
bind custom-key your-command
```

**WezTerm**
```lua
-- ~/.config/wezterm/wezterm.lua
{ key = "key", mods = "MODS", action = act.YourAction }
```
